# Security Policy

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 1.0.x   | Yes       |

## Reporting a Vulnerability

If you discover a security issue in TETRA OS, please report it responsibly:

1. **Do not** open a public GitHub issue for security vulnerabilities.
2. Email the maintainer with a detailed description, steps to reproduce, and impact assessment.
3. Allow reasonable time for a fix before public disclosure.

**Contact:** open a private security advisory on [GitHub](https://github.com/wcalmels/tetra-os/security/advisories/new) or contact Walter Calmels through the repository.

We aim to acknowledge reports within 48 hours and provide a remediation timeline when possible.
